# quantum-plateau-bypass

量子AIのBarren Plateau（砂漠化）問題に対して、「戦わずに回避する」アプローチで取り組みました。量子回路出力を固定粒度で離散化する翻訳層（STE実装）を導入することで、量子-古典間のインターフェースで情報エントロピーを意図的に削減。過学習の抑制と勾配の安定伝播を同時に実現しています。4qubit/8qubit・最大6層の深い回路でも勾配消失は確認されず、Irisデータセットで正解率100%、Wineデータセット（13次元）で94%、乳がんデータ（30次元）92％を達成しました。
最先端の量子コンピューティングでは実機で30Qubitの量子機械学習の実績が存在していますが、層数についての言及がなく一応意味があるかと思い、公開しています。

## 1. はじめに

本アーキテクチャは、量子機械学習における「砂漠化（勾配消失 / Barren Plateau）」問題の根本原因に対する直感的な仮説から出発し、AIとの対話的な検証を通じて設計された独自のハイブリッドモデルです。従来の量子ML研究とは異なる視点からアプローチしているため、一部の説明において標準的な理論用語とは異なる表現を用いている場合があります。
個人のシミュレーション環境における物理的な計算資源の制約上、当方で提供できる実証データは「4層・4Qubitモデルにおいて砂漠化を回避し、正答率100％を達成した」という段階までとなります。これをさらに多層化・大規模化した場合に真に機能するかどうかは、現環境では断言できません。
これ以上のスケーリングと限界の探求については、より高度なハードウェア環境や専門機関を持つ研究者の方々の手で検証・発展していただきたく、本ロジックをオープンソースとして公開することにしました。

## 3. なぜこの設計なのか

本アーキテクチャでは、量子機械学習における砂漠化問題の根本原因を、**「量子回路の出力空間における連続自由度が極めて高く、有限のサンプル（測定回数）条件下では、頻度差として有意な特徴を安定観測できないこと（情報エントロピーの実質的発散）」**にあると定義しました。
つまり、勾配消失は「原因」ではなく「結果」です。
「サンプリング回数を増やす」「学習率を調整する」「データ量を増やす」といった従来の古典的なアプローチでは、有限時間内に十分な頻度統計を形成できず、結果としてパラメータの更新方向が不安定化していくと考えています。

そこで着想を得たのが、画像フォーマット（WebPなど）における**「知覚上重要ではない差異を統合する損失圧縮」**の思想です。

人間の目でPNGとWebPの微細な差異を認識できないのと同様に、量子出力の `1.000000` と `0.999999` の差にも、実際の学習においては実質的な意味（有意な情報差）が存在しない場合があります。近傍値を「翻訳層（STE実装）」によってあえて離散化・統合することで、統計的な頻度を強制的に集約し、有限の観測下でも特徴を安定して抽出できる状態へ再構成します。

これは単なる情報の削減（ダウンサンプリング）ではありません。

* **粒度が細かすぎる場合：** 1つの情報（状態）あたりの責務が過剰に分散し、「特徴そのもの」よりも「特徴を区別するための境界情報（ノイズ）」が支配的になります。結果として有限サンプル内での安定観測が不可能になります。
* **粒度が粗すぎる場合：** 本質的な特徴の差異まで潰れてしまい、識別能力そのものが失われます。

> **【本設計のコア思想】**
> 粗すぎる粒度も、細かすぎる粒度も、極端化すればどちらもAIにとっては“認識不能”になる。近傍状態を適度に統合することで、1特徴あたりの責務を回復させ、**「有限時間・有限観測下で真に学習可能な情報密度」**へとチューニングする。

```text
ディレクトリ構成イメージ
project_folder/
 ├── quantum_classifier.py  # コアモジュール
 ├── config.json            # 設定ファイル
 └── main.py                # メインスクリプト
main.py での実装例
外部スクリプトからは、「設定を読み込む」 ➔ 「学習をぶん回す」 ➔ 「一番良いモデルで推論する」 という3ステップだけで完結します。

Python
from pennylane import numpy as np
from sklearn.datasets import load_breast_cancer

# 独自の量子モジュールから必要な部品をインポート
from quantum_classifier import load_config, launch_quantum_hunt, HybridQuantumClassifier

def main():
    # ==========================================
    # 1. データの準備 (パイプライン等から持ってくる想定)
    # ==========================================
    cancer = load_breast_cancer()
    X_scaled = (cancer.data - np.mean(cancer.data, axis=0)) / np.std(cancer.data, axis=0)
    y = cancer.target
    
    X_train, y_train = X_scaled[:400], y[:400]
    X_test, y_test = X_scaled[400:], y[400:]

    # ==========================================
    # 2. 学習フェーズ (並列ハンティング)
    # ==========================================
    # 設定ファイルのロード
    config = load_config("config.json")
    
    # ハンティング開始（一番精度が高かった重みと粒度を自動で返してくる）
    print("学習を開始します...")
    best_weights, best_step = launch_quantum_hunt(
        config=config, 
        X_train=X_train, 
        y_train=y_train, 
        X_test=X_test, 
        y_test=y_test, 
        log_dir="logs"
    )

    # ==========================================
    # 3. 推論フェーズ (未知のデータへの予測)
    # ==========================================
    # 最適モデルのインスタンスを立ち上げ、最高の重みをセット
    model = HybridQuantumClassifier(config)
    model.weights = best_weights

    # 例：テストデータの最初の5件を予測してみる
    print(f"\n採用されたStep Size: {best_step}")
    for i in range(5):
        raw_pred = model.predict(X_test[i], best_step)
        binary_pred = 1 if raw_pred >= 0.5 else 0
        print(f"データ[{i}] | 予測: {binary_pred} (実値: {y_test[i]}) | 生の出力値: {raw_pred:.4f}")

if __name__ == "__main__":
    main()
```
